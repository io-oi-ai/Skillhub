# Product Landing Page Copy

## Skill Overview

A Landing Page is the first impression for acquiring users for your indie product. Excellent Landing Page copy captures visitor attention in seconds, communicates product value, and drives user action. Based on best practices for high-converting copy, this skill helps you generate all Landing Page section copy at once, covering the complete page structure from Hero section to FAQ.

## Input Requirements

Please provide the following information:

- **Product Name and One-Liner**: What is your product
- **Target Users**: Who will use your product and what pain points do they currently face
- **Core Features**: The product's 3-5 main functions or features
- **Differentiation**: What is your unique advantage compared to competitors
- **Pricing Model**: Free, paid, Freemium, or other model
- **Desired User Action**: Sign up, download, purchase, join waitlist, etc.
- **Brand Tone** (optional): Professional/serious, casual/friendly, geeky/technical, etc.

## Output Content

### 1. Hero Section

- **Main Headline**: One sentence that captures user attention (6-12 characters), hitting pain points or value
- **Subheadline**: Supplementary explanation of how the product solves the problem (15-25 characters)
- **CTA Button Copy**: Text for the main call-to-action button (e.g., "Get Started Free", "Try Now")
- Provide 3 alternative options for selection

### 2. Social Proof Bar

- Suggested data metrics worth displaying (e.g., user count, positive reviews, time saved)
- Trust signals worth showcasing (e.g., media coverage, partner integrations, security certifications)

### 3. Pain Point Resonance Section

- 3-4 core pain point descriptions for target users
- Use "Are you also..." resonance-style phrasing
- Guide toward the product as a solution

### 4. Feature Section

- For each core feature, generate:
  - Feature title (3-6 characters)
  - Feature description (1-2 sentences, emphasize user benefits rather than technical details)
  - Suggested icon type to pair with

### 5. How It Works Usage Flow

- 3-4 clear usage steps
- Each step includes a step title and brief description
- Emphasize the "simple and easy to use" feeling

### 6. Pricing Section Copy

- Generate corresponding copy based on pricing model
- Price anchoring strategy suggestions
- Feature comparison descriptions for each pricing tier
- CTA button copy

### 7. User Testimonials / Social Proof Section

- Suggested angles for collecting testimonials and template
- Example testimonial structure (if no real testimonials available yet for early stage)

### 8. FAQ Section

- 6-8 frequently asked questions and answers
- Cover product features, pricing, security, technical support, etc.

### 9. Bottom CTA Section

- Final conversion copy: one-sentence value summary + action button
- Create urgency or emphasize zero-risk

### 10. SEO Metadata

- **Title Tag**: No more than 60 characters
- **Meta Description**: No more than 160 characters
- **OG Title** and **OG Description**
- Suggested target keyword list

## High-Converting Copy Formulas

The following copy formulas can help you refine each section:

- **PAS Formula**: Problem → Agitate → Solution
- **AIDA Formula**: Attention → Interest → Desire → Action
- **Before-After-Bridge**: Show before/after contrast; the product is the connecting bridge
- **Feature → Benefit → Outcome**: Feature → Benefit → Final Result. Users don't care about features; they care about results

## Usage Tips

- **The Headline is Most Important**: 80% of people only read the headline; spend the most time perfecting your Hero section headline
- **Speak Human Language**: Avoid industry jargon and abstract concepts; describe value in users' language
- **One Page, One Goal**: A Landing Page should push one core action; don't make users choose
- **Mobile-First**: Over 60% of traffic comes from mobile; ensure copy reads well on small screens
- **A/B Testing**: Generate multiple alternative versions and let real data decide the final copy
- **Numbers Trump Words**: Use concrete numbers instead of vague descriptions; "saves 3 hours/week" is more effective than "saves lots of time"
